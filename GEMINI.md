# GEMINI.md - Project Context for tsr-app

This project, **tsr-app**, is a modern profit-tracking dashboard for Shopee sellers, optimized for AI-assisted development through the **Everything Claude Code (ECC)** system.

## Project Overview

*   **Type:** Next.js Dashboard Application (E-commerce Analytics).
*   **Main Technologies:**
    *   **Framework:** Next.js 16.2.2 (App Router) with React 19.2.4.
    *   **Database & Auth:** Supabase via `@supabase/ssr`.
    *   **Styling:** Tailwind CSS 4 (using CSS variables for theming).
    *   **Icons:** Lucide-React.
    *   **Data Processing:** `papaparse` (CSV) and `xlsx` (Excel).
    *   **Visualization:** Recharts.
*   **Architecture:** Server-first approach using React Server Components (RSC) and Supabase SSR clients for secure data fetching.

## Building and Running

*   **Development:** `npm run dev` - Starts the development server at `http://localhost:3000`.
*   **Build:** `npm run build` - Creates a production build.
*   **Start:** `npm run start` - Starts the production server.
*   **Lint:** `npm run lint` - Runs ESLint.

## Directory Structure

*   `src/app/`: Next.js App Router (pages, layouts, and API routes).
    *   `/dashboard`: Main analytics interface.
    *   `/login` & `/auth`: Authentication flow.
*   `src/components/`: Reusable UI components.
*   `src/lib/`: Core business logic and utilities.
    *   `/supabase`: Server and Client Supabase factories.
    *   `profit.ts`: Margin and profit calculation logic.
*   `src/middleware.ts`: Auth enforcement and session management.
*   `everything-claude-code/`: Specialized agentic workflows and skills (ECC integration).

## Development Conventions

    *   Use `src/lib/supabase/client.ts` for Client Components.
*   **Theming:** Use CSS variables (e.g., `var(--primary)`, `var(--surface)`) for styling to maintain compatibility with the Tailwind 4 setup.
*   **Commits:** Use conventional commits (`feat`, `fix`, `refactor`, `docs`, `test`, `chore`).

## Security Checklist

*   **Auth:** Always verify user session in Server Components before fetching sensitive data.
*   **Secrets:** Never hardcode API keys. Use `.env.local` for `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY`.
*   **Data Validation:** Validate all external data (CSV/Excel imports) using libraries like `papaparse` before processing.
